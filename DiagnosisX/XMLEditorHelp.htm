2. Основные функции работы с XML-документом

XML-редактор предоставляет средства для создания корректного (правильно построенного) XML-документа - древесной структуры, состоящей из узлов и атрибутов. Документы можно создавать, сохранять и загружать.

Редактор поддерживает открытие документа из командой строки (принимает один параметр с путем к файлу документа). Таким образом, xml-файлы можно связать с редактором с помощью команды контекстного меню Windows Open with... и вызывать его по правой кнопке.

Создание, редактирование, удаление и другие операции над элементами XML-дерева выполняются с помощью контекстного меню или главного меню, кроме того, ко всем часто используемым функциям привязаны горячие клавиши.

2.1. Создание, редактирование узлов

Редактор позволяет создавать новые узлы и либо помещать их над выбранным узлом, либо добавлять их после существующих. Можно создать узел, являющийся дочерним по отношению к выбранному. Любому узлу можно поменять название, для этого достаточно на выбрать узел и тыкнуть на него мышкой. Имена узлов должны являться допустимой последовательностью XML NMTOKEN. Имена узлов могут совпадать.

Узел может иметь либо дочерние узлы, либо значение. Приписывание значения узлу приводит к удалению его дочерних узлов. Значение представляет собой произвольную текстовую строку и соответствует последовательности CDATA.

Для выполнения операций по созданию узлов предусмотрены горячие клавиши Shift+Right Arrow (добавление дочернего узла), Shift+Up Arrow (вставка нового узла), Shift+Down Arrow (добавление нового узла).

2.2. Копирование, вставка и удаление узлов

Узел или поддерево можно скопировать в буфер обмена редактора (не windows), а затем вставить его в любое место дерева перед выбранным узлом, за исключением корневого узла. Буфер обмена может одновременно хранить только одно поддерево. Удаление узла приводит к каскадному удалению соответствующего поддерева, то есть всех его дочерних узлов.

Для выполнения этих операций предусмотрены горячие клавиши Ctrl+C (копирование), Ctrl+V (вставка), Ctrl+X (удаление).

2.3. Создание, редактирование, удаление атрибутов

У любого узла может быть произвольное количество атрибутов. Имена атрибутов должны соответствовать последовательности NMTOKEN. Атрибуты узла должны быть уникальны в рамках одного узла.

3. Инструменты

3.1. Проверка XML-документа на допустимость

Если XML-документ формируется в соответствии с какой-либо схемой документа (DTD), то для возможности проверки допустимости документа в нем должна быть помещена DTD-схема или ссылка на файл с DTD-схемой. Для выполнения проверки на допустимость следует выбрать пункт главного меню "Проверить допустимость" либо нажать на кнопку "DTD CHK" на панели инструментов.

3.2. Контекстная подсказка для выбранного узла в виде фрагмента DTD

Контекстная подсказка работает только в случае существования DTD-схемы  в документе (либо ссылки на файл с DTD-схемой). При выборе произвольного узла дерева, в DTD-схеме по названию узла ищется фрагмент с описаниями узла ELEMENT и ATTLIST. В случае нахождения фрагмента он выводится внизу экрана. Для неописанных в DTD-схеме узлов подсказка не выводится.

3.3. Просмотр атрибутов в дереве узлов

По умолчанию атрибуты в дереве узлов  не показаны, однако при наведении мыши на узел всплывает подсказка, содержащая в себе список атрибутов узла. Для отображения атрибутов в самом дереве следует включить опцию "Показывать атрибуты в дереве" (нажать кнопочку с деревом). Редактирование атрибутов в дереве узлов невозможно, однако название узла изменить можно.

3.4. Просмотр XML-документа в окне Internet Explorer

XML-документ можно просмотреть через Internet Explorer не выходя из редактора. Для этого нужно сохранить изменения и выбрать пункт меню "Открыть в Internet Explorer" (нажать кнопочку с логотипом IE).

3.5. Поиск по документу

Редактор предоставляет возможность поиска текста в XML-документе. Поиск осуществляется с выбранного узла вниз по дереву. Поиск можно настроить на независимый просмотр имен узлов, значений узлов, имен и значений атрибутов. Кроме того, можно указать обязательность полного совпадения искомой строки с просматриваемым элементом дерева. При успешном поиске выделяется узел, в котором была найдена искомая строка.

Для осуществления поиска предусмотрены горячие клавише Ctrl+F (начать поиск) и F3 (продолжить поиск) .
